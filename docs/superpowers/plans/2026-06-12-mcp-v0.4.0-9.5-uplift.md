# uniprot-link v0.4.0 ">9.5/10" Uplift — Implementation Plan

> Historical record — this document records the design or plan as of its date. Current behavior is
> defined by implemented code, standards, release evidence, and tests.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the 12 residual bugs + token/latency/consistency gaps from
`docs/mcp-assessment.md` so a rubric re-run scores every dimension 9–10 (> 9.5/10).

**Architecture:** Incremental, test-first edits to an existing FastMCP 3.4.2 +
SPARQL service. Envelope/error changes are centralized in `mcp/envelope.py` +
`next_commands.py`; data fixes live in `services/` (split `queries.py` into a
package first to stay under the 600-line cap); typed output lands in a new
`mcp/schemas.py`. Every SPARQL change is re-validated live.

**Tech Stack:** Python 3.12, FastMCP 3.4.2, httpx, pydantic, pytest + respx,
ruff, mypy (strict), `uv`. Endpoint: UniProt QLever SPARQL (release 2026_01).

**Spec:** `docs/superpowers/specs/2026-06-12-mcp-v0.4.0-9.5-uplift-design.md`.

**Conventions (verified):**
- Tests: `tests/unit/*.py` use `make_select_json(vars, rows)` and the
  `service_factory` fixture (backed by `FakeSparqlClient`, which routes by the
  **first query-substring match**). Multi-query tools (`features`/`variants`/
  `diseases`/`cross_references`/`go_terms`) issue an `entry_exists_ask` ASK
  (`a up:Protein`) **and** a data query — tests must route the ASK substring to a
  `{"boolean": true}` body or the tool raises `not_found`.
- Live re-validation: `python research/verify_queries.py` (per CLAUDE.md).
- Gate before any "done": `make ci-local` (format-check, lint-ci, **lint-loc**,
  typecheck, test-fast).
- Modern typing (`list[str]`, `str | None`), ASCII, Ruff/mypy-strict clean.
- Commit style mirrors the repo: `feat(...)`/`fix(...)`/`refactor(...)` +
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.

**Branch:** create `feat/v0.4.0-uplift` off `main` before Task 0.

```python
class InvalidInputError(SparqlClientError):
    """A tool/service argument failed validation before any query ran."""

    def __init__(
        self,
        message: str,
        field: str | None = None,
        *,
        allowed: list[str] | None = None,
        hint: str | None = None,
    ) -> None:
        """Initialise with the offending field and optional recovery data."""
        super().__init__(message)
        self.field = field
        self.allowed = allowed
        self.hint = hint
```

```python
def protein_not_found_recovery(value: str) -> list[dict[str, Any]]:
```
